# 数据层设计

## 概述

数据层负责与 Electron 主进程通信，完成文件读写、标记文件序列化/反序列化、图片加载等任务。数据层作为应用与底层系统的桥梁，确保数据的持久化存储和高效加载。

## 核心功能

1. **文件操作**
   - 标记文件的新建、打开、保存、另存为
   - 图片文件的添加、删除
   - 导出图片为 PNG 格式

2. **序列化/反序列化**
   - 标记文件的 JSON 序列化与反序列化
   - 标注数据的序列化与反序列化

3. **图片加载**
   - 图片文件的读取和加载
   - 图片元数据的获取

4. **Electron 通信**
   - 与主进程的 IPC 通信
   - 处理文件对话框的显示

## 实现方案

### 1. Electron IPC 通信

使用 Electron 的 IPC (Inter-Process Communication) 机制实现渲染进程与主进程之间的通信。

**主要通信通道**：

- `file:new`: 新建标记文件
- `file:open`: 打开标记文件
- `file:save`: 保存标记文件
- `file:save-as`: 另存为标记文件
- `file:add-image`: 添加图片
- `file:remove-image`: 删除图片
- `file:export-png`: 导出 PNG

### 2. 标记文件格式

标记文件采用 JSON 格式存储，包含以下结构：

```json
{
  "version": "1.0",
  "images": [
    {
      "id": "image-1",
      "name": "example.jpg",
      "path": "/path/to/image.jpg",
      "width": 1920,
      "height": 1080,
      "annotations": [
        {
          "id": "annotation-1",
          "type": "horizontal-line",
          "startX": 100,
          "startY": 200,
          "endX": 300,
          "endY": 200,
          "createdAt": "2026-03-19T10:00:00Z",
          "updatedAt": "2026-03-19T10:05:00Z"
        }
      ]
    }
  ],
  "createdAt": "2026-03-19T09:00:00Z",
  "updatedAt": "2026-03-19T10:05:00Z"
}
```

### 3. 序列化器

**标记文件序列化器**：

- 将应用状态转换为 JSON 格式
- 将 JSON 格式转换回应用状态

**标注序列化器**：

- 处理不同类型标注的序列化
- 确保标注数据的完整性和一致性

### 4. 错误处理

- 文件读写错误处理
- 图片加载错误处理
- 序列化/反序列化错误处理

### 5. 性能优化

- 大文件的异步处理
- 图片的懒加载
- 序列化/反序列化的缓存机制

## 与其他层的交互

- **与业务逻辑层**：通过 IPC 事件和回调函数进行数据交换
- **与表现层**：通过状态管理库（如 Redux）传递数据变更

## 安全考虑

- 文件路径验证，避免访问敏感目录
- 图片文件类型验证，防止恶意文件加载
- 标记文件内容验证，确保数据完整性
