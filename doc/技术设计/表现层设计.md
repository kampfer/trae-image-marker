# 表现层设计

## 组件结构概览

根据需求文档，Trae Image Marker 应用采用上中下三部分垂直布局，表现层组件结构如下：

```
App
├── ThemeProvider (主题提供者)
│   ├── StoreProvider (状态管理提供者)
│   │   ├── MenuBar (顶部菜单栏)
│   │   ├── TabsBar (标签页栏)
│   │   │   └── WorkSpace (工作区)
│   │   │       ├── Toolbar (工具栏)
│   │   │       └── PixiCanvas (画布区域)
│   │   │           ├── ImageContainer (图片容器)
│   │   │           └── Annotations (标注层)
│   │   └── Footer (底部状态栏)
```

**菜单可用性计算逻辑**：

菜单项的可用性由Redux Selector计算，根据以下状态：

- `isFileOpened`: 是否有标记文件打开
- `isFileModified`: 文件是否已修改
- `activeImageId`: 当前选中的图片ID
- `hasAnnotations(imageId)`: 图片是否有标注
- `hasSelectedAnnotations(imageId)`: 图片是否有选中的标注
- `canUndo(imageId)`: 是否可以撤销
- `canRedo(imageId)`: 是否可以重做
- `zoomLevel`: 当前缩放级别
- `minZoom`/`maxZoom`: 最小/最大缩放限制

**状态传递实现方案**：

由于项目不使用 hooks，我们使用类组件结合 Redux 的 `connect` 高阶组件来实现状态传递。

##### 2. 按钮状态

按钮有两种视觉状态：

- **默认状态**：正常显示，未激活
- **激活状态**：高亮显示（type="primary"），表示当前正在执行或已开启

激活状态的触发方式分两类：

| 类型 | 说明 | 示例 |
|------|------|------|
| 异步激活（async） | 命令触发后进入激活状态，命令完成（Promise resolve/reject）后恢复默认 | 标注工具、图片操作按钮 |
| 状态切换激活（toggle） | 按钮激活状态与 Redux 状态绑定，状态为 true 时激活，false 时恢复默认 | 辅助线开关 |

##### 4. 激活状态实现逻辑

```typescript
// 异步激活：点击后设置 activeKey，命令完成后清除
const [activeKey, setActiveKey] = useState<string | null>(null);

const handleClick = async (button: ToolbarButton) => {
  if (button.activationType === 'async') {
    setActiveKey(button.key);
    try {
      await dispatch(button.action);
    } finally {
      setActiveKey(null);
    }
  } else {
    dispatch(button.action);
  }
};

// 渲染时根据 activationType 判断按钮激活状态
const getButtonType = (button: ToolbarButton): 'primary' | 'default' => {
  if (button.activationType === 'async') {
    return activeKey === button.key ? 'primary' : 'default';
  }
  // toggle 类型：通过 activeSelector 从 Redux state 读取，经 connect 注入后判断
  return button.activeSelector?.(reduxState) ? 'primary' : 'default';
};
```

### 标注创建流程

1. 用户从顶部菜单或悬浮工具栏选择标注类型
2. 系统进入标注创建模式
3. 用户在画布上点击确定起点
4. 用户移动鼠标确定终点
5. 用户再次点击完成标注创建
6. 系统将标注添加到标注层

### 标注编辑流程

1. 用户点击标注进行选择
2. 系统显示标注的选中状态和控制点
3. 用户拖拽标注整体移动或拖拽控制点调整标注
4. 用户按 Delete 键或通过右键菜单删除标注

### 图片操作流程

1. 用户从顶部菜单或悬浮工具栏选择图片操作
2. 系统执行相应的图片操作（缩放、旋转等）
3. 系统更新图片显示和底部状态栏信息

### 文件操作流程

1. 用户从顶部菜单选择文件操作
2. 系统弹出相应的对话框（如打开文件对话框）
3. 用户完成操作后，系统执行相应的文件操作
4. 系统更新应用状态和界面显示

## 组件状态管理

### StoreProvider 状态管理

**全局状态**：

- 当前打开的标记文件路径
- 当前激活的标签页
- 所有标签页的图片和标注数据
- 撤销/重做历史
- 应用配置信息

**状态管理方案**：

- 使用 Redux 或 Zustand 管理全局状态
- 定义清晰的 action 和 reducer
- 支持状态持久化到本地存储

### 组件局部状态

**画布状态**：

- 当前图片的缩放比率
- 当前图片的旋转角度
- 当前选中的标注
- 当前正在创建的标注
- 辅助线显示状态

**标注状态**：

- 标注的几何信息（坐标、尺寸、角度等）
- 标注的样式（颜色、线宽、字体等）
- 标注的选中状态

## 组件实现要点

### 性能优化

- 使用 React.memo 优化组件渲染
- 使用 useCallback 和 useMemo 优化函数和计算值
- 标注渲染使用 Canvas API 提高性能
- 图片缩放和旋转使用 CSS transform 提高性能

### 可扩展性

- 组件设计遵循模块化原则，便于添加新功能
- 标注类型设计为可扩展的，便于添加新的标注类型
- 工具类设计为可扩展的，便于添加新的工具

### 用户体验

- 提供直观的工具提示
- 支持键盘快捷键
- 提供流畅的动画效果
- 确保操作响应迅速
