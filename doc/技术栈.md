# Trae Image Marker 技术栈文档

## 项目概述

Trae Image Marker 是一个基于 Electron 的跨平台桌面应用程序，用于图像标记功能。

## 核心技术栈

### 桌面应用框架

- **Electron 40.4.1**
  - 跨平台桌面应用开发框架
  - 使用 Chromium 和 Node.js 构建
  - 支持Windows、macOS和Linux平台

- **Electron Forge 7.11.1**
  - Electron应用构建和打包工具
  - 提供统一的构建和分发流程
  - 支持多种打包格式

### 前端框架

- **React 19.2.4**
  - 用户界面库
  - 组件化开发模式
  - 虚拟DOM高效渲染

- **React DOM 19.2.4**
  - React的DOM渲染器
  - 用于浏览器环境中的React渲染

- **React Redux 9.2.0**
  - React官方Redux绑定库
  - 状态管理解决方案
  - 提供Provider和hooks API

### UI组件库

- **Ant Design 6.3.0**
  - 企业级React UI组件库
  - 提供丰富的组件和设计规范
  - 支持主题定制

### 开发语言

- **TypeScript 5.9.3**
  - JavaScript的超集
  - 提供静态类型检查
  - 增强代码可维护性和开发体验

### 构建工具

- **Vite 5.4.21**
  - 新一代前端构建工具
  - 快速的开发服务器
  - 优化的生产构建
  - 支持热模块替换(HMR)

## 开发工具

### 代码质量工具

- **ESLint 8.57.1**
  - JavaScript/TypeScript代码检查工具
  - 发现代码中的问题和潜在错误
  - 强制执行代码风格规范

- **Prettier 3.8.1**
  - 代码格式化工具
  - 统一代码风格
  - 支持多种编程语言

- **TypeScript ESLint 5.62.0**
  - TypeScript的ESLint插件
  - 提供TypeScript特定的ESLint规则

- **ESLint Plugin Import 2.32.0**
  - 支持import语句的ESLint规则
  - 自动排序import语句
  - 检查import路径的正确性

### Git工作流工具

- **Husky 9.1.7**
  - Git钩子管理工具
  - 在Git操作时自动执行脚本
  - 确保代码质量

- **lint-staged 16.2.7**
  - 对暂存文件运行linters的工具
  - 只检查即将提交的文件
  - 提高Git提交速度

## 打包和分发

### 打包工具

- **Electron Forge Makers**
  - 支持多种打包格式：
    - **Squirrel** - Windows安装程序
    - **ZIP** - 通用压缩包
    - **DEB** - Debian包
    - **RPM** - Red Hat包

- **Electron Fuses 1.8.0**
  - Electron安全配置工具
  - 增强应用安全性
  - 配置Electron运行时行为

### 其他工具

- **Electron Squirrel Startup 1.0.1**
  - Windows Squirrel启动处理
  - 处理Windows自动更新

## 开发特性

### CSS支持

- **CSS Modules**
  - 模块化CSS支持
  - 避免类名冲突
  - 支持局部作用域样式

### 代码规范

- **Import自动排序**
  - 自动整理import语句
  - 按组排序（builtin、external、internal等）
  - 字母顺序排列

- **Git提交自动格式化**
  - 提交时自动运行Prettier
  - 确保代码风格一致
  - 通过Husky和lint-staged实现

### 代码格式化配置

- **Prettier配置**
  - 采用前端社区最常用的代码风格
  - 使用单引号、分号、2空格缩进
  - 每行最大100字符
  - ES5尾随逗号
  - 箭头函数参数始终使用括号
  - LF换行符（Unix风格）

- **ESLint忽略配置**
  - 忽略依赖目录和构建输出
  - 忽略配置文件和文档
  - 忽略日志和临时文件

## 项目配置

### TypeScript配置

- 支持JSX语法（React组件）
- 启用严格类型检查
- 配置模块解析策略
- 支持ESNext特性

### Vite配置

- 主进程配置（vite.main.config.ts）
- 预加载脚本配置（vite.preload.config.ts）
- 渲染进程配置（vite.renderer.config.ts）
- CSS Modules配置

### ESLint配置

- TypeScript支持
- Import规则
- React规则
- 代码质量检查
- Import自动排序规则
- 支持ES5和ESNext特性

### Prettier配置

- 前端社区常用代码风格
- 单引号、分号、2空格缩进
- 每行最大100字符
- ES5尾随逗号
- 箭头函数参数始终使用括号
- LF换行符（Unix风格）
- JSX使用双引号

### Git钩子配置

- Husky管理Git钩子
- pre-commit钩子自动运行lint-staged
- 提交前自动格式化代码
- 只检查暂存的文件

## 可用脚本

### 开发脚本

- `npm start` - 启动开发服务器
- `npm run lint` - 运行代码检查
- `npm run format` - 格式化代码

### 构建脚本

- `npm run package` - 打包应用
- `npm run make` - 构建安装包
- `npm run publish` - 发布应用

## 项目文件结构

```
trae-image-marker/
├── .husky/                 # Git钩子配置
│   └── pre-commit         # 提交前自动执行脚本
├── .eslintrc.json         # ESLint配置
├── .eslintignore          # ESLint忽略文件
├── .prettierrc            # Prettier配置
├── .prettierignore        # Prettier忽略文件
├── doc/                   # 文档目录
│   └── 技术栈.md          # 技术栈文档
├── src/                   # 源代码目录
│   ├── main.ts           # 主进程
│   ├── preload.ts        # 预加载脚本
│   ├── renderer.tsx      # 渲染进程（React应用）
│   └── index.css         # 全局样式
├── forge.config.ts        # Electron Forge配置
├── index.html             # HTML入口文件
├── package.json           # 项目配置和依赖
├── tsconfig.json          # TypeScript配置
├── vite.main.config.ts    # 主进程Vite配置
├── vite.preload.config.ts # 预加载脚本Vite配置
└── vite.renderer.config.ts # 渲染进程Vite配置
```
