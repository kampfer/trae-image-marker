# Trae Image Marker 技术设计文档

## 1. 项目概述

### 1.1 项目背景

Trae Image Marker 是一个基于 Electron 的跨平台桌面应用程序，专为建模师设计的图片标注和比例测量工具。

### 1.2 项目目标

- 为建模师提供直观、高效的图片标注和比例测量界面
- 支持多种标注类型，满足建模师测量参考图比例关系的需求
- 实现标注数据的持久化存储，方便建模师保存和复用测量数据
- 支持跨平台运行（Windows、macOS、Linux）
- 提供良好的用户体验和操作便捷性，提高建模师的工作效率

### 1.3 目标用户

主要用户群体为建模师，包括角色建模师、场景建模师、道具建模师和动画建模师。

## 2. 系统架构

### 2.1 整体架构

采用 Electron 多进程架构，分为主进程、预加载脚本和渲染进程。

### 2.2 技术分层

- 表现层：React 组件、Ant Design UI 组件、PixiJS 绘制
- 业务逻辑层：Redux Actions & Reducers、标注创建/编辑/删除逻辑、图片缩放/旋转逻辑、撤销/重做逻辑
- 数据层：文件读写、标记文件序列化/反序列化、图片加载、本地文件系统、标记文件 (JSON 格式)

## 6. 组件架构

### 6.1 组件层次结构

```
App
└── Provider (Redux Provider)
    └── ConfigProvider (Ant Design 主题配置)
        └── Layout
            ├── Header
            │   ├── FileMenu (文件菜单)
            │   ├── EditMenu (编辑菜单)
            │   ├── AnnotationMenu (标注菜单)
            │   └── ViewMenu (视图菜单)
            ├── Workspace
            │   ├── TabsBar
            │   ├── PixiCanvas (画布)
            │   └── Toolbar (浮动工具栏)
            └── Footer
```

### 6.2 组件设计原则

- 容器组件：负责数据获取和状态管理，连接 Redux Store
- 展示组件：负责渲染 UI，通过 props 接收数据和回调事件
- 布局组件：负责页面布局，使用 Ant Design Layout 组件
- 功能组件：实现特定功能，如标注绘制、图片缩放等

### 6.4 Store 架构

#### 6.4.1 Store 与组件的关联

| 组件       | 关联的 Store 状态         | 主要职责                         |
| ---------- | ------------------------- | -------------------------------- |
| Header     | ui, settings              | 菜单操作、主题切换、设置管理     |
| TabsBar    | markerFile                | 标签页管理、图片切换             |
| PixiCanvas | drawing, view, markerFile | 标注绘制、画布操作、视图控制     |
| Toolbar    | drawing, view, history    | 工具选择、缩放控制、撤销/重做    |
| Footer     | ui, view                  | 状态显示、缩放显示、画布尺寸显示 |

#### 6.4.2 容器组件与展示组件

- **容器组件**：连接 Redux Store，处理数据逻辑
  - 负责从 Store 中获取状态
  - 触发 Actions 来更新状态
  - 将状态和回调传递给展示组件

- **展示组件**：纯 UI 组件，无状态或轻状态
  - 通过 props 接收数据和回调
  - 专注于 UI 渲染
  - 不直接与 Store 交互

#### 6.4.3 数据流

```
Store → 容器组件 → 展示组件
        ↓           ↓
        Actions ← 事件处理
```
