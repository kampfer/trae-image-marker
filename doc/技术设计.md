# Trae Image Marker 技术设计文档

## 1. 项目概述

### 1.1 项目背景

Trae Image Marker 是一个基于 Electron 的跨平台桌面应用程序，专为建模师设计的图片标注和比例测量工具。建模师在进行 3D 建模时，经常需要参考真实照片或设计图，通过测量参考图中的比例关系来确保模型的准确性和真实感。

### 1.2 项目目标

- 为建模师提供直观、高效的图片标注和比例测量界面
- 支持多种标注类型，满足建模师测量参考图比例关系的需求
- 实现标注数据的持久化存储，方便建模师保存和复用测量数据
- 支持跨平台运行（Windows、macOS、Linux）
- 提供良好的用户体验和操作便捷性，提高建模师的工作效率

### 1.3 目标用户

主要用户群体为建模师，包括：

- 角色建模师：测量人物照片中的身体比例关系
- 场景建模师：测量建筑照片或设计图中的尺寸比例和角度关系
- 道具建模师：测量物体照片中的尺寸、角度和比例关系
- 动画建模师：测量参考图中的动作角度和比例关系

## 2. 系统架构

### 2.1 整体架构

采用 Electron 多进程架构，分为以下三个主要进程：

```
┌─────────────────────────────────────────────────────────┐
│                    主进程 (Main Process)                 │
│  - 文件系统访问                                          │
│  - 窗口管理                                              │
│  - 应用生命周期管理                                      │
│  - IPC 通信协调                                          │
└─────────────────────────────────────────────────────────┘
                            ↕ IPC
┌─────────────────────────────────────────────────────────┐
│                  预加载脚本 (Preload Script)             │
│  - 安全的 API 暴露                                       │
│  - 上下文隔离                                            │
│  - IPC 桥接                                              │
└─────────────────────────────────────────────────────────┘
                            ↕ IPC
┌─────────────────────────────────────────────────────────┐
│                渲染进程 (Renderer Process)               │
│  - React 应用                                            │
│  - 用户界面渲染                                          │
│  - 标注绘制和交互                                        │
│  - 状态管理 (Redux)                                      │
└─────────────────────────────────────────────────────────┘
```

### 2.2 进程职责

#### 2.2.1 主进程

- **文件操作**：处理文件的读取、写入、删除等操作
- **窗口管理**：创建和管理应用窗口
- **IPC 通信**：与渲染进程进行进程间通信
- **应用生命周期**：处理应用的启动、退出等事件
- **菜单管理**：创建和管理应用菜单
- **对话框管理**：处理文件选择、保存等对话框

#### 2.2.2 预加载脚本

- **API 暴露**：将主进程的功能安全地暴露给渲染进程
- **上下文隔离**：确保渲染进程无法直接访问 Node.js API
- **类型定义**：提供 TypeScript 类型定义

#### 2.2.3 渲染进程

- **UI 渲染**：使用 React 渲染用户界面
- **图片和标注绘制**：使用 PixiJS 在 WebGL/Canvas 上绘制图片和标注
  - 使用 PixiJS 的 Stage 和 Container 管理渲染场景
  - 使用 Sprite 加载和显示图片
  - 使用 Graphics 绘制各种标注（线段、量角器等）
  - 支持图片的缩放、旋转和平移变换
  - 利用 PixiJS 的事件系统处理标注的交互
- **用户交互**：处理鼠标、键盘等用户输入
- **状态管理**：使用 Redux 管理应用状态
- **业务逻辑**：实现标注的创建、编辑、删除等功能

```typescript
const drawingSlice = createSlice({
  name: 'drawing',
  initialState: initialDrawingState,
  reducers: {
    setTool: (state, action: PayloadAction<ToolType>) => {
      state.currentTool = action.payload;
      state.isDrawing = false;
      state.currentAnnotation = null;
    },
    startDrawing: (state, action: PayloadAction<Partial<Annotation>>) => {
      state.isDrawing = true;
      state.currentAnnotation = action.payload;
    },
    updateDrawing: (state, action: PayloadAction<Partial<Annotation>>) => {
      if (state.currentAnnotation) {
        state.currentAnnotation = { ...state.currentAnnotation, ...action.payload };
      }
    },
    finishDrawing: (state, action: PayloadAction<Annotation>) => {
      state.isDrawing = false;
      state.currentAnnotation = null;
      state.selectedAnnotationId = action.payload.id;
    },
    cancelDrawing: (state) => {
      state.isDrawing = false;
      state.currentAnnotation = null;
    },
    selectAnnotation: (state, action: PayloadAction<string>) => {
      state.selectedAnnotationId = action.payload;
    },
    deselectAnnotation: (state) => {
      state.selectedAnnotationId = null;
    },
  },
});
```

```typescript
const viewSlice = createSlice({
  name: 'view',
  initialState: initialViewState,
  reducers: {
    setScale: (state, action: PayloadAction<number>) => {
      state.scale = action.payload;
      state.fitToWindow = false;
    },
    setRotation: (state, action: PayloadAction<number>) => {
      state.rotation = action.payload;
    },
    setOffset: (state, action: PayloadAction<{ x: number; y: number }>) => {
      state.offsetX = action.payload.x;
      state.offsetY = action.payload.y;
    },
    zoomIn: (state) => {
      state.scale = Math.min(state.scale * 1.2, 10);
      state.fitToWindow = false;
    },
    zoomOut: (state) => {
      state.scale = Math.max(state.scale / 1.2, 0.1);
      state.fitToWindow = false;
    },
    fitToWindow: (state, action: PayloadAction<{ width: number; height: number }>) => {
      const { width, height } = action.payload;
      const scaleX = width / 1920;
      const scaleY = height / 1080;
      state.scale = Math.min(scaleX, scaleY);
      state.fitToWindow = true;
    },
    actualSize: (state) => {
      state.scale = 1;
      state.fitToWindow = false;
    },
  },
});
```

#### 5.4.4 History Reducer

```typescript
const historySlice = createSlice({
  name: 'history',
  initialState: initialHistoryState,
  reducers: {
    pushHistory: (state, action: PayloadAction<MarkerFile>) => {
      if (state.present) {
        state.past.push(state.present);
      }
      state.present = action.payload;
      state.future = [];
      if (state.past.length > state.maxSize) {
        state.past.shift();
      }
    },
    undo: (state) => {
      if (state.past.length > 0) {
        state.future.push(state.present!);
        state.present = state.past.pop()!;
      }
    },
    redo: (state) => {
      if (state.future.length > 0) {
        state.past.push(state.present!);
        state.present = state.future.pop()!;
      }
    },
    clearHistory: (state) => {
      state.past = [];
      state.future = [];
    },
  },
});
```

### 6.2 组件设计原则

#### 6.2.1 组件分类

1. **容器组件 (Container Components)**
   - 负责数据获取和状态管理
   - 不直接渲染 UI，而是渲染展示组件
   - 连接 Redux Store

2. **展示组件 (Presentational Components)**
   - 负责渲染 UI
   - 通过 props 接收数据
   - 通过 props 回调事件

3. **布局组件 (Layout Components)**
   - 负责页面布局
   - 使用 Ant Design Layout 组件

4. **功能组件 (Functional Components)**
   - 实现特定功能
   - 如标注绘制、图片缩放等

#### 6.2.2 组件命名规范

- 使用 PascalCase 命名组件
- 组件文件名与组件名一致
- 样式文件名与组件名一致，使用 `.module.css` 后缀

#### 6.2.3 组件职责划分

- **单一职责**：每个组件只负责一个功能
- **高内聚低耦合**：组件内部逻辑紧密，组件之间依赖最小化
- **可复用性**：组件设计考虑复用性

3. **类型集中管理**：类型定义统一放在 `types` 目录
4. **服务层分离**：业务逻辑放在 `services` 目录

```typescript
class ImageCache {
  private cache: Map<string, HTMLImageElement> = new Map();
  private maxSize = 10;

  async loadImage(filePath: string): Promise<HTMLImageElement> {
    if (this.cache.has(filePath)) {
      return this.cache.get(filePath)!;
    }

    const image = await this.loadImageFromFile(filePath);

    if (this.cache.size >= this.maxSize) {
      const firstKey = this.cache.keys().next().value;
      this.cache.delete(firstKey);
    }

    this.cache.set(filePath, image);
    return image;
  }

  clear(): void {
    this.cache.clear();
  }
}
```
